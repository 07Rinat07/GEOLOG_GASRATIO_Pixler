# Редактор LAS

Версия: 0.7.6

В главной панели программы доступен отдельный раздел **«Редактор»** и кнопка
**«Редактор LAS»** (`Ctrl+Alt+E`). Он объединяет создание, ручное редактирование,
исправление, изменение шага, вставку кривых и сращивание LAS-файлов.

## Безопасность исходников

Редактор не перезаписывает исходные LAS. Операции исправления, ресэмплинга,
вставки и сращивания создают новый независимый dataset и новый файл `.las`.
Исходный путь, кодировка, SHA-256 и выполненная операция записываются в метаданные
или manifest результата.

## Операции

- **Создать новый LAS** — диапазон, шаг, индекс MD/TVD/TVDSS, LAS 1.2/2.0 и NULL.
- **Открыть LAS** — импорт одного или нескольких файлов в отдельные рабочие скважины.
- **Редактировать таблицу** — правка значений, заголовков и кривых; изменения остаются
  в рабочей модели до экспорта новой копии.
- **Исправить убывающую глубину** — синхронный разворот глубины, индексов и всех кривых.
- **Изменить шаг глубины** — новая копия с произвольным положительным шагом, например
  0,2; 0,5 или 1 м. Интерполяция не соединяет NULL и большие разрывы.
- **Вставить данные из LAS** — выбор отдельных кривых внешнего файла и сопоставление
  по глубине в новую копию рабочего LAS.
- **Срастить LAS** — объединение глубинных участков двух загруженных файлов с политикой
  перекрытия: сохранить текущие значения, предпочесть источник или оставить обе кривые.
- **Сохранить текущий как новый LAS** — обычный экспорт рабочей копии.

## Нестандартные LAS

Редактор поддерживает:

- убывающую глубину и отрицательный `STEP`;
- кодировки UTF-8, Windows-1251 и CP866, определённые импортёром;
- повторяющиеся мнемоники, которые `lasio` представляет как `GK:1`, `GK:2`;
- кириллические и содержащие разделители мнемоники.

Исходные имена сохраняются для диагностики, а для выходного LAS автоматически
предлагаются переносимые имена: `GK:1 → GK_1`, `КС, ННК/ДСР → KS_NNK_DSR`.
Пользователь может изменить предложенную мнемонику до сохранения.

## Вставка кривых

1. Откройте LAS-приёмник.
2. Откройте **Редактор → Вставить данные из LAS**.
3. Выберите внешний LAS.
4. Проверьте исходные и выходные мнемоники, единицы и описания.
5. Снимите флажки с ненужных кривых.
6. Укажите имя нового LAS.
7. Нажмите **«Вставить и сохранить копию»**.

Убывающий внешний файл разворачивается только в памяти. Значения переносятся на
сетку приёмника в общей области глубин; вне перекрытия записывается NULL.

## Сращивание

Откройте оба LAS, выберите файл-приёмник в дереве проекта и запустите
**Редактор → Срастить LAS**. Выберите второй dataset, политику перекрытия и имя
нового файла. Оба исходных файла останутся неизменными.
