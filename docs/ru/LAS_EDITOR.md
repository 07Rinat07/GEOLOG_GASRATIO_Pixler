# Редактор LAS

Версия функции: 0.7.6. Раздел **«Редактор»** и команда **«Редактор LAS»** (`Ctrl+Alt+E`)
объединяют создание LAS, ручное редактирование, исправление глубины, изменение шага, вставку
кривых и сращивание файлов.

## Безопасность исходных файлов

Редактор не перезаписывает исходный LAS. Исправление, ресэмплинг, вставка и сращивание создают
новый независимый dataset и новый файл `.las`. Исходный путь, кодировка, SHA-256 и выполненная
операция сохраняются в metadata или manifest результата.

Изменения таблицы сначала находятся в рабочем dataset проекта. **Ctrl+S** сохраняет проект, но
не создаёт новый LAS-файл. Для обмена с другой программой используйте экспорт новой копии LAS.

## Основные операции

- **Создать новый LAS** — диапазон, шаг, индекс MD/TVD/TVDSS, LAS 1.2/2.0 и NULL.
- **Открыть LAS** — импорт одного или нескольких файлов в отдельные рабочие скважины.
- **Редактировать таблицу** — изменение значений, заголовков и кривых с Undo/Redo.
- **Исправить убывающую глубину** — синхронный разворот глубины, индексов и всех кривых.
- **Изменить шаг глубины** — новая копия с положительным шагом, например 0,2; 0,5 или 1 м.
- **Вставить данные из LAS** — выбор кривых внешнего файла и сопоставление по глубине.
- **Срастить LAS** — объединение участков с явной политикой перекрытия.
- **Сохранить текущий как новый LAS** — экспорт рабочей копии в отдельный файл.

## Нестандартные и старые LAS

Поддерживаются убывающая глубина, отрицательный `STEP`, UTF-8, Windows-1251 и CP866,
повторяющиеся мнемоники и кириллические названия. Исходные имена сохраняются для диагностики,
а для выходного LAS предлагаются переносимые варианты: `GK:1 → GK_1`,
`КС, ННК/ДСР → KS_NNK_DSR`. Перед экспортом пользователь может изменить предложенное имя.

Интерполяция при смене шага не соединяет NULL и большие разрывы. Перед подтверждением проверяйте
диапазон, количество строк, единицы, NULL и предупреждения Import Review.

## Вставка кривых

1. Откройте LAS-приёмник.
2. Выберите **Редактор → Вставить данные из LAS**.
3. Выберите внешний LAS.
4. Проверьте исходные и выходные мнемоники, единицы и описания.
5. Снимите флажки с ненужных кривых.
6. Укажите путь новой копии LAS.
7. Нажмите **«Вставить и сохранить копию»**.

Убывающий внешний файл разворачивается только в памяти. Значения переносятся на сетку приёмника
в общей области глубин; вне перекрытия записывается NULL. Исходные файлы не изменяются.

## Сращивание

1. Откройте оба LAS.
2. Выберите dataset-приёмник в дереве проекта.
3. Запустите **Редактор → Срастить LAS**.
4. Выберите второй dataset и политику перекрытия:
   - сохранить текущие значения;
   - предпочесть значения источника;
   - оставить обе кривые под разными мнемониками.
5. Укажите имя нового файла и подтвердите операцию.

Оба исходных файла остаются неизменными. После создания результата проверьте начало/конец,
перекрытие, NULL, единицы и заголовки.

## Сохранение и контрольное открытие

- **Ctrl+S** сохраняет рабочую модель внутри проекта.
- **Экспорт LAS** создаёт отдельный переносимый файл.
- Закрытие проекта без сохранения отменяет изменения рабочей таблицы после последнего `Ctrl+S`.
- После важного редактирования сохраните проект, экспортируйте копию LAS и повторно откройте оба
  результата для проверки количества строк, индекса и ключевых кривых.
