# Примечания к выпуску 0.7.27 — удаление и изоляция аннотаций по формам

## Исправлено удаление

- команда **«Удалить аннотацию»** в контекстном меню теперь сравнивает результат `QMessageBox` по значению, а не по идентичности Python-объекта;
- после подтверждения объект удаляется из модели скважины, исчезает с планшета и создаёт одну операцию Undo;
- в окне редактирования существующей аннотации добавлена отдельная кнопка **«Удалить аннотацию»**;
- общий менеджер аннотаций обновляет планшет сразу после добавления, изменения, дублирования, удаления, Undo и Redo, без закрытия окна;
- после удаления очищается выбор и отключаются действия для уже несуществующего объекта.
- контекстное меню сравнивает выбранный `QAction` по значению и передаёт удаление через единый контроллер модели;

## Аннотации больше не переходят в другие формы

- каждой аннотации присваивается стабильная область вида текущего набора данных и формы планшета;
- комментарии, выноски, значения кривых и изображения отображаются только в форме, в которой были созданы;
- применение другой формы скрывает чужие аннотации, а возврат к исходной форме восстанавливает их;
- изменение состава дорожек внутри текущей формы не меняет область аннотаций;
- сохранение текущего планшета как пользовательской формы переносит связанные аннотации в область новой формы;
- экранный планшет, PDF и прямой Masterlog-рендерер используют одинаковую фильтрацию области.
- после переключения формы отложенный сигнал со старым идентификатором не может изменить или удалить объект предыдущей формы; контроллер повторно проверяет `scope_id` перед операцией;

## Совместимость проектов

Старые аннотации без области вида автоматически привязываются к сохранённой форме планшета при первом открытии версии 0.7.27. Исходный текст, стиль, геометрия, глубина, время, кривая, изображения и настройки печати не изменяются.

## Проверка

Добавлены проверки:

- CRUD и Undo/Redo после удаления;
- стабильная область аннотаций при изменении формы;
- отсутствие аннотации в другой форме;
- возврат аннотации при повторном открытии исходной формы;
- миграция старого объекта без `scope_id`;
- сохранение `annotation_scope_id` в компоновке планшета;
- наличие кнопки удаления в одиночном редакторе;
- отсутствие загрузки well-global аннотаций до восстановления сохранённой формы.
