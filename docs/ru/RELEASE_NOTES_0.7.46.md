# GEOLOG GASRATIO@Pixler 0.7.46 — восстановление импорта LAS и диагностика

Аварийная тестовая сборка от 24 июля 2026 года. Project format остаётся **v20**.

## Исправленная регрессия

В 0.7.45 новый overlay инженерной сетки передавал в PySide6 обычное целое значение `0`
вместо типизированного `Qt.MouseButton.NoButton`. На актуальных версиях PySide6 исключение
возникало при первом построении планшета уже после успешного чтения и подтверждения LAS.
Dataset успевал зарегистрироваться, но рабочая область оставалась чёрной и создавалось
впечатление, что файл не открылся.

В 0.7.46:

- используется корректный `Qt.MouseButton.NoButton`;
- сбой overlay-сетки больше не блокирует доступ к данным: включается безопасная axis-grid
  замена и ошибка записывается в журнал;
- если построение планшета или браузера кривых всё же завершается ошибкой, программа открывает
  импортированный dataset в табличном режиме восстановления;
- предупреждения Import Review не считаются ошибкой и не запрещают открытие файла;
- неожиданные исключения перехватываются отдельно для каждого файла и не выходят в Qt event loop.

## Диагностический отчёт

Импорт теперь собирает единый отчёт по этапам `read`, `parse`, `policy`, `review`, `register`
и `present`. Для каждой проблемы сохраняются уровень, код, файл, понятное описание, действие
для исправления, контекст, тип исключения и полный traceback.

Отчёт можно скопировать или сохранить из диалога. При блокирующей ошибке техническая копия
автоматически записывается атомарно в каталог данных приложения, чтобы её можно было передать
разработчику даже после закрытия окна.

## Устойчивость LAS

- значения кривых читаются по физическому номеру столбца, поэтому повторяющиеся vendor-мнемоники
  не подменяют друг друга;
- повреждение отдельного канала создаёт предупреждение и исключает только этот канал, а не весь LAS;
- добавлен regression-сценарий на 9 847 строк и 73 смешанных кириллических/латинских канала;
- default layout ограничивает число первоначально отображаемых кривых, чтобы большой LAS не
  перегружал первый кадр.

## Совместимость и проверка

Project format, form schema, tablet layout, рабочие формы, обозначения и история ежедневного
наращивания LAS не изменены.

- focused import/recovery/diagnostics suite: **76 passed**;
- доступная headless-регрессия: **1011 passed, 4 skipped, 3 deselected**;
- `compileall` выполнен; wheel 0.7.46 собран успешно;
- в контейнере отсутствуют `PySide6`, `pyqtgraph` и `lasio`, поэтому обязательный Windows smoke-test
  реального LAS и первого Qt-кадра остаётся перед stable.
