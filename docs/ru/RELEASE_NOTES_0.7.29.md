# Примечания к выпуску 0.7.29 — единая граница импортных jobs

## Что изменено

- `ImportJobController` перенесён из Qt-пакета в `services/import_jobs.py`;
- создан единый `DatasetImportJobExecutor` для CSV/Excel, пакетного LAS и регистрации Paradox;
- LAS strict/compatible/manual policy, диагностические предупреждения, ручной пропуск и
  определение убывающей глубины больше не выполняются в `MainWindow`;
- lossless LAS source document и import report регистрируются через один project-session port;
- Paradox dataset и отдельная скважина добавляются через ту же границу после успешного
  завершения фонового импорта;
- отменённый, отклонённый или ошибочный файл не оставляет частично зарегистрированный dataset;
- счётчик загруженных LAS теперь показывает фактически успешно зарегистрированные файлы;
- LAS-адаптер загружается лениво, поэтому routing и CSV/Excel jobs не требуют `lasio` при импорте модуля.

## Совместимость

Формат проекта, формат планшета, LAS/DB-файлы и последовательность действий пользователя не
изменены. Исходные файлы по-прежнему открываются только для чтения.

## Проверка

Расширенный регрессионный набор для import jobs, адаптеров и project-session boundary:
105 тестов пройдено, 3 платформенных теста пропущены; `compileall` и сборка wheel завершены без ошибок. Полный Ruff/mypy/pytest gate требуется повторить в установленном окружении
со всеми зависимостями.
