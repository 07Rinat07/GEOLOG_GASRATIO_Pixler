# Release notes 0.7.26 — исправление типизированного плана DB → LAS

## Исправленная ошибка

На Windows `QComboBox.currentData()` может вернуть значение `StrEnum` как обычную строку. Ручной план, созданный из пакетного конвертера, доходил до импортёра с `classification="time_with_depth"` или `duplicate_depth_policy="keep_all"` в строковом виде. При записи метаданных код обращался к `.value`, поэтому операция завершалась сообщением:

```text
'str' object has no attribute 'value'
```

`ParadoxImportPlan` теперь является единой типизированной границей. При создании он:

- преобразует строковые значения Qt и JSON в `DatasetClassification` и `DuplicateDepthPolicy`;
- проверяет `active_role`, `NULL`, язык, поля индекса и сопоставления каналов;
- отклоняет неизвестные значения до начала чтения и экспорта;
- гарантирует, что внутренний импортёр работает только с нормализованным планом.

Окно ручной настройки дополнительно выполняет явное преобразование значений Qt перед созданием плана.

## Диагностика пакетной операции

Пакетный конвертер теперь добавляет к ошибке этап операции:

- чтение DB;
- анализ каналов;
- подготовка плана импорта;
- создание набора данных;
- запись LAS;
- повторное открытие LAS.

Вместо необъяснимого внутреннего сообщения пользователь видит, на каком этапе остановился конкретный файл.

## Шаг 0,4 м и стандарт 0,2 м

Ошибка `.value` не связана с шагом глубины. LAS 2.0 не требует обязательного шага 0,2 м. Если строки исходного DB реально записаны через 0,4 м, экспортируемый `STEP` должен быть 0,4 м. Стандартный серверный шаг GeoScape 0,2 м сохраняется отдельно в метаданных.

Для получения сетки 0,2 м необходимо явно выполнить ресэмплинг в редакторе LAS и выбрать метод интерполяции. Пакетный конвертер не создаёт искусственные промежуточные строки и не подменяет `STEP` молча.

## Совместимость

- исходные DB/PX/TV/FAM открываются только для чтения;
- формат проекта и схема аннотаций не изменены;
- профили 0.7.16–0.7.25 продолжают загружаться;
- строковые значения старых и новых профилей нормализуются одинаково;
- сборка остаётся тестовой до реального Windows-теста `DB → LAS → повторное открытие`.
