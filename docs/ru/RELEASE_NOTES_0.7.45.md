# GEOLOG GASRATIO@Pixler 0.7.45 — рабочие формы и ежедневное наращивание LAS

Тестовая сборка от 23 июля 2026 года. Project format: v20. Form schema: v6. Tablet layout: v15.

## Пользовательский сценарий

- графические колонки сразу показывают инженерную сетку даже при скрытых осях pyqtgraph;
- у каждой колонки независимо настраиваются вертикальная и горизонтальная сетка, основные и промежуточные деления, прозрачность и печать сетки;
- шапка кривой показывает название, начало и конец шкалы, единицу и linear/log режим;
- двойной щелчок по строке параметра в шапке открывает настройки;
- отдельно сохраняются цвет кривой, цвет названия и цвет линии под названием;
- пользовательская форма сохраняет порядок и ширину колонок, кривые, шкалы, сетку, подписи, видимый вертикальный интервал, источник и номер ревизии;
- комментарии, изображения и глубинные значки остаются в стабильной области `dataset + form`;
- 19 заводских обозначений, включая фоновый и пластовый газ, поставляются как прозрачные PNG с плотной обрезкой;
- команда «Ежедневно нарастить LAS…» добавляет только новые строки в явно выбранный dataset текущей скважины;
- DEPTH и TIME, разные единицы, типы индекса, скважины и схемы кривых не смешиваются;
- совпадающее перекрытие пропускается, конфликтующее перекрытие блокирует операцию без частичного изменения;
- повторный импорт того же файла определяется по SHA-256 и становится безопасной операцией без изменений;
- история каждого dataset хранит отдельные записи источника, диапазона и количества добавленных строк;
- один проект поддерживает несколько независимых DEPTH/TIME datasets, каждый со своими layout/form/annotations;
- при создании нового глубинного LAS шаг по умолчанию установлен 0,2 м и остаётся редактируемым.

## Совместимость

- project format v19 автоматически мигрирует в v20 с пустой историей наращивания для каждого dataset;
- старые form schema 0–5 мигрируют в v6;
- старые tablet layout 1–14 мигрируют в v15;
- исходный LAS не перезаписывается: проект хранит составной dataset и audit history, а формы/значки/комментарии сохраняются отдельно от строк LAS.
