# Импорт CSV и TXT

Команда «Файл → Импортировать CSV/TXT...» открывает безопасный предпросмотр таблицы.
Пользователь выбирает кодировку, разделитель и индексную колонку до создания dataset.

Поддерживаются:

- автоматическое определение `,`, `;`, TAB и `|`;
- явный выбор этих разделителей;
- UTF-8 с BOM, UTF-8, CP1251 и Latin-1;
- десятичная точка и десятичная запятая при непересекающемся разделителе;
- NULL-маркеры: пустая строка, NULL, NA, N/A, NAN и `-999.25`;
- единицы в заголовках вида `C1 [%]` или `ROP [m/h]`.
- индексная колонка ISO-8601 с `Z`, явным UTC offset или без зоны.

Заголовки должны быть непустыми и уникальными без учёта регистра. Каждая строка обязана
иметь одинаковое число колонок, а текущий этап принимает числовые каналы. Ошибка содержит
название колонки и номер строки. Исходный файл открывается только для чтения и не изменяется.

Если выбранный индекс содержит ISO-8601, aware-значения преобразуются в UTC, а исходный
offset сохраняется в `DatasetIndex.timezone`. Время без зоны не считается UTC и хранится с
`timezone=None`. Остальные импортируемые каналы пока должны быть числовыми.

Составные колонки DATE+TIME и пользовательские форматы уже поддерживаются normalization-
ядром, но их сопоставление в CSV-мастере остаётся следующим UI-инкрементом.
