# Обзор GeoData как функционального референса

Изучено пользовательское руководство информационно-измерительной системы GeoData
(ООО «Геотехсервис», 2022, 66 страниц). Руководство используется только для анализа
рабочих сценариев; текст, изображения и оформление в проект не копируются.

## Полезные сценарии для развития

1. Статистика числовых кривых по выбранному глубинному блоку.
2. Именованные шаблоны компоновки треков, включая цветной и чёрно-белый варианты.
3. Примечания и инженерные маркеры, привязанные к глубине.
4. Разные вертикальные масштабы для отдельных интервалов глубины.
5. Индивидуальный стиль, диапазон и способ отображения каждой кривой.
6. Табличный редактор с тем же выделенным интервалом, что и графический планшет.
7. Предварительный просмотр и проверка сводной диаграммы перед экспортом/печатью.

## Реализовано по итогам обзора

Добавлен безопасный расчёт статистики по видимому интервалу планшета: количество
валидных точек, минимум, максимум и среднее для каждой числовой кривой. Расчёт не
изменяет dataset и вынесен из Qt в независимый модуль.

## Навигация и рабочая область

Руководство GeoData подтверждает два полезных для планшета решения: данные прокручиваются одной вертикальной полосой справа для всех колонок, а рабочий масштаб выбирается отдельно от полного диапазона скважины. Кроме того, рабочий экран и группы инструментов настраиваются, чтобы освобождать место под диаграмму.

В текущем срезе GEOLOG применены эти принципы без копирования интерфейса: длинный LAS открывается в читаемом рабочем окне, все колонки прокручиваются одной камерой, а боковые панели запускаются из компактных сворачиваемых реек.

## Принятые правила для рабочих форм LAS

По руководству рабочий экран должен быть не пустой схемой, а сразу отображать данные
открытого файла; состав колонок, кривые, диапазоны и оформление затем настраиваются и
сохраняются как именованный экран. Для GEOLOG это зафиксировано следующими правилами:

- после открытия LAS базовая форма материализуется по реальным кривым текущего dataset;
- применение формы считается успешным только после фактического построения curve items;
- отсутствие стиля у кривой не должно останавливать отрисовку: применяется безопасный стиль;
- настройки колонок и кривых сохраняются в пользовательской копии формы;
- ошибки построения формы не скрываются пустым планшетом, а выдаются в диагностике.

Реализация использует собственные модели и код; из GeoData заимствуется только проверенная
логика рабочего процесса.
