# Импорт Excel

Команда «Файл → Импортировать Excel...» открывает XLS/XLSX/XLSM только для чтения. В диалоге
можно выбрать лист, номер строки с заголовками и индексную колонку. Предпросмотр ограничен,
а полная книга читается `openpyxl` в режимах `read_only=True` и `data_only=True`: формулы не
исполняются внутри приложения. Если книга содержит формулы, LibreOffice Calc пересчитывает
их в headless-режиме с отдельным временным профилем, после чего импортируются результаты.

Заголовки должны быть непустыми и уникальными. Пустые строки пропускаются, пустые ячейки
преобразуются в NULL. Числовые, ISO-8601 и нативные Excel date/time значения проходят через
ту же нормализацию и доменную модель, что CSV/TXT. Оригинальная книга не изменяется.

Если дата и время находятся в разных колонках, включите «Объединить индексную колонку DATE
с отдельной TIME», выберите TIME и задайте `strptime`-форматы. Можно указать IANA timezone
(`Asia/Oral`) или фиксированный offset (`UTC+05:00`). Обе исходные колонки объединяются в
один DATETIME-индекс и не дублируются как кривые.

Поддержка Excel входит в основные зависимости приложения и устанавливается автоматически:

```bash
pip install -e .
```

Старый бинарный `.xls` безопасно конвертируется LibreOffice Calc во временный XLSX. Для XLS
и пересчёта формул установите системный компонент `sudo apt install libreoffice-calc`.
Временные файлы удаляются после импорта, оригинал не изменяется.
Для локальной portable-установки путь к `soffice` можно задать переменной окружения
`GEOLOG_LIBREOFFICE`; каталог `.dev-tools/` исключён из Git.

Макросы XLSM намеренно не запускаются: VBA является исполняемым кодом и не нужен для чтения
табличных результатов. Импорт создаёт dataset, поэтому макросы в проект не переносятся.
