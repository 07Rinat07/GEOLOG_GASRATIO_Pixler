# План проекта GEOLOG GASRATIO@Pixler

## Правила структуры

- `README.md` остаётся в корне и содержит знакомство, запуск и автора.
- вся подробная документация хранится строго в `docs/`;
- реализация ведётся небольшими рабочими инкрементами;
- каждый инкремент сопровождается тестами и обновлением документации;
- исходный LAS не перезаписывается скрытно.

## Реализованные этапы

### 0.1–0.2 — базовое ядро

- модели проекта, скважины, набора данных и кривой;
- импорт LAS;
- проектная сессия;
- базовые Gas Ratio;
- безопасное атомарное сохранение;
- реестр методик Pixler;
- тесты вычислительного ядра.

### 0.4 — Tablet Engine Foundation

- отдельный модуль `tablet`;
- модель `TrackDefinition` и `TabletLayout`;
- базовые типы треков;
- многотрековый экран планшета;
- синхронизация треков по общей оси глубины;
- горизонтальная прокрутка;
- построение базовой компоновки по открытому LAS;
- трек глубины;
- газовый трек;
- треки отдельных кривых;
- переключение dataset из Project Explorer;
- инспектор выбранного трека;
- модель глубинного viewport для будущих zoom/pan и виртуализации.

## Следующий инкремент 0.5 — Tablet Track Manager

1. Диалог выбора кривых для нового трека.
2. Изменение ширины трека мышью и через инспектор.
3. Скрытие, удаление и перемещение треков.
4. Сохранение компоновки планшета в проекте.
5. Линейная и логарифмическая шкалы X.
6. Настройка диапазона каждой кривой.
7. Легенда кривых и единиц.
8. Оптимизированное отображение видимого глубинного диапазона.
9. Сохранение и восстановление текущего интервала глубины.
10. Тесты компоновки, синхронизации и сериализации.

## Последующие этапы

- 0.6: Interactive Curve Editor — карандаш, Undo/Redo и пересчёт зависимостей.
- 0.7: Gas/Pixler Engine — подтверждённые методики и паспорта формул.
- 0.8: Geology — литология, шламограмма, ЛБА, стратиграфия и керн.
- 0.9: Catalog Studio — породы, литотипы, узоры и обозначения.
- 0.10: Multi-Well Correlation — несколько скважин, GR и реперы.
- 0.11: Print Studio — формы, шапки, PDF и системный принтер.
- 1.0: стабильный формат проекта, миграции, восстановление и установщик.
