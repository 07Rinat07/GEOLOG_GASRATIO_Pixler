# Инкремент 0.5 — Tablet Track Manager

## Цель

Дать пользователю безопасные команды управления составом и геометрией планшета,
а также подготовить стабильный формат хранения его компоновки.

## Реализованный срез

- выбор одной или нескольких кривых при создании трека;
- изменение ширины выбранного трека с минимальным значением 80 px;
- перемещение выбранного трека влево и вправо;
- скрытие, восстановление и удаление треков;
- отметка сессии как изменённой после команд менеджера;
- версионируемое преобразование `TabletLayout` в JSON-совместимый словарь и обратно;
- строгая проверка типов, версии формата и уникальности `track_id`;
- версионированный формат файла проекта с совместимым чтением старых JSON;
- сохранение отдельной компоновки планшета для каждого dataset;
- открытие проекта и восстановление выбранного dataset и его layout;
- модульные тесты операций модели, round-trip и ошибочных входных данных.

## Оставшаяся работа

- реализовать изменение ширины мышью и редактирование через инспектор;
- добавить линейную/логарифмическую шкалу и диапазоны кривых;
- сохранять видимый интервал глубины;
- добавить Qt-тесты пользовательских команд и синхронизации.

Формат проекта версии 2 хранит domain-данные и словарь `tablet_layouts`, связанный по
`dataset_id`. Файлы прежнего формата без поля версии читаются как legacy-проекты с пустым
набором сохранённых компоновок; при следующем сохранении они записываются в актуальном формате.
