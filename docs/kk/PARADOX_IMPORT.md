# GeoScape / Borland Paradox DB импорты — 0.7.29

## 0.7.29 нұсқасындағы import тіркеуі

DB оқу және түрлендіру бұрынғыдай тоқтатуға болатын жұмыс ағынында орындалады. Сәтті нәтиже `DatasetImportJobExecutor` шекарасына беріледі, ол dataset пен жаңа ұңғыманы project-session арқылы тіркейді. `MainWindow` енді жоба моделін тікелей өзгертпейді. Қате, тоқтату немесе пайдаланушы бас тартуы жобаға жартылай dataset қоспайды; бастапқы DB өзгермейді.

## 0.7.26 пакеттік түрлендіру түзетуі

Windows-та Qt тізім мәндері кейде кәдімгі жол ретінде қайтарылады. Пакеттік файлды қолмен баптағаннан кейін бұл бұрын `'str' object has no attribute 'value'` қатесін тудырды. Енді импорт жоспары жіктеуді, қайталанатын тереңдік ережесін, белсенді индексті, NULL және тілді бірден қалыпқа келтіреді. Бұл 0,4/0,2 м қадамына байланысты емес: LAS нақты `STEP=0.4` мәнін қабылдайды; 0,2 м торы тек айқын ресэмплинг арқылы жасалады. Қате жолында оқу, талдау, жоспарлау, импорт, LAS жазу немесе қайта ашу кезеңі де көрсетіледі.

## Жауап беретін терезе және қауіпсіз жабу

Бинарлық блоктарды оқу, талдау және Dataset құру жұмыс ағындарында орындалады. Арналар, алдын ала қарау және диагностика кестелері Qt таймері арқылы қысқа бөліктермен толтырылады, сондықтан интерфейс қайта сызылып, жабу командасын қабылдайды. Толтыру кезінде қымбат `ResizeToContents` қолданылмайды.

Жоғарыда ағымдағы файл, алты кезеңнің бірі, жалпы пайыз, өңделген саны және өткен уақыт көрсетіледі. Төменгі панельде «Болдырмау/Жабу», «LAS сақтау» және «Редакторда ашу» әрқашан көрінеді. Операция кезінде жабу қауіпсіз тоқтатуды сұрайды; бастапқы DB өзгермейді.

## Мақсаты

Импорттаушы GeoScape/Borland Paradox кестесін қолданыстағы GEOLOG GASRATIO@Pixler `Dataset` моделіне түрлендіреді. Расталғаннан кейін кәдімгі кесте, графиктер, LAS редакторы, жоба, біріктіру, планшет және баспа компоненттері пайдаланылады; бөлек редактор жасалмайды.

## Ашу

**Файл → Импорттау → GeoScape / Paradox DB** командасын, әмбебап импортты немесе `.db` drag-and-drop қолданыңыз. Формат кеңейтіммен ғана анықталмайды: SQLite сигнатурасы және Paradox бинарлық құрылымы тексеріледі. DB/PX/TV/FAM тек оқу режимінде ашылады және ешқашан қайта жазылмайды.

`BLData.db` таңдалса, бағдарлама регистрге тәуелсіз `BLData.PX`, `BLData.TV`, `BLData.FAM` файлдарын іздейді. Қосымша файл жоқ болса, негізгі DB импорты жалғасады және ескерту көрсетіледі.

## Талдау терезесі

Терезе форматты, нұсқаны, өлшемді, жазба/өріс санын және табылған жиынтықты көрсетеді. Арналар кестесінде импортты қосуға, LAS мнемоникасын, атауын және бірлігін өзгертуге болады. Алдын ала қарау тек алғашқы және соңғы 20 жолды жүктейді. Оқу, талдау және құжат жасау фондық ағында орындалып, прогресс пен бас тартуды қолдайды.

Тереңдік пен уақыт кандидаттары толымдылық, монотондылық, диапазон, қадам тұрақтылығы, атау және кері секіру бойынша бағаланады. Екіұшты индекс автоматты қолданылмайды. OLE/Delphi, Unix және салыстырмалы уақыт қолдау табады. `TIME` секунд индексі жасалып, бастапқы сан `<арна>_RAW` қисығында сақталады.

## Арналар, профильдер және сапа

Белгісіз `Sxxx` болжанбайды: бастапқы мнемоника сақталады, бірлік бос, сипаттама бастапқы арна ретінде беріледі. JSON сөздігі расталған сәйкестіктерді сақтайды, пайдаланушы сәйкестігі басым. Профиль схема SHA-256 сигнатурасымен қорғалады және тек толық сәйкес құрылымға қолданылады.

Бос арналар/жолдар, NaN/Infinity, өте үлкен мәндер, статистикалық ауытқулар, қайталанатын/кері/теріс тереңдік, секірулер және уақыт хронологиясы тексеріледі. Әдепкіде қайталанатын жолдар жойылмайды; бірінші, соңғы, орташа және медиана ережелері тек пайдаланушы таңдағанда қолданылады. Хаттамада схема қолтаңбасы, таңдалған индекстер, NULL, сұрыптау, импортталған/өткізілген арналар мен жолдар саны, диагностика деңгейлері және қолданылған түзетулер сақталады.

## Тереңдіктің стандартты және нақты қадамы

GeoScape серверлік тіркеуінің стандартты номиналды қадамы **0,2 м**. Импорттаушы оны таңдалған тереңдік арнасының нақты қадамынан бөлек көрсетеді. Берілген `BLData.db` жолдары негізінен **0,4 м** сайын орналасқан — файл сервердің басқа баптауымен жазылған болуы мүмкін.

LAS тақырыбындағы `STEP` нақты бар деректер торын сипаттайды. Сондықтан 0,4 м жолдар үшін жалған `STEP=0.2` жазылмайды. 0,2 м туынды торын жасау үшін құжатты ашып, **LAS редакторы → Тереңдік қадамын өзгерту…** командасын анық орындап, интерполяция әдісін таңдаңыз. Бастапқы DB және бастапқы импортталған жинақ өзгермейді.

Пакеттік түрлендіргіш те осы келісімді қолданады: бастапқы қадам 0,2 м болса `STEP=0.2`, ал 0,4 м болса `STEP=0.4` жазылады. Аралық жолдар жасырын қосылмайды. **«Тереңдік LAS торы»** өрісінде **GeoScape 0,2 м** туынды торын анық таңдауға болады: тереңдік сұрыпталады, қайталанған тереңдіктің соңғы жолы сақталады және сандық арналар NULL үзілістерін біріктірмей сызықтық интерполяцияланады. **«Дереккөздегідей»** қауіпсіз әдепкі таңдау болып қалады. Алдымен уақытша LAS жасалып, кірістірілген reader толық индексті, `STRT/STOP/STEP`, арналар жиынын және олардың мәндерін салыстырады. Тексеру сәтті өткенде ғана соңғы файл сақталады; қате болса уақытша файл жойылып, бұрынғы LAS ауыстырылмайды.

## LAS, TIME → DEPTH және пакет

Тереңдік LAS бірінші индекс ретінде `DEPT.M`, уақыт LAS `TIME.SEC` пайдаланады; `STRT/STOP/STEP` нақты деректерден есептеледі. TIME → DEPTH бастапқы деректі өзгертпей жаңа туынды жиынтық жасайды және бірінші/соңғы/орташа/медиана/min/max/ең жақын/сызықтық әдістерді ұсынады.

**Құралдар → DB → LAS пакеттік түрлендіру** файлдар мен каталогтарды, рекурсивті іздеуді, профильді, нәтиже маскасын, қайта жазудан қорғауды, прогресті, бас тартуды және JSON журналын қолдайды. Екіұшты файл профильсіз автоматты түрде болжанбайды; ол «Баптау қажет» күйін алады және жолдан тікелей бапталады.

## Тексерілген үлгілер

Paradox 7.x `NUMBER` және `LONG` өрістері нақты расталды: `BLData.db` — 3488 жазба/70 өріс, `D250.db` — 1739/101. Alpha, Date, Short, Logical, Time, Timestamp, AutoIncrement, BCD және Bytes/Blob үшін шектеулі декодер бар, бірақ олар берілген үлгілерде болмағандықтан қосымша нақты файлдарда тексерілуі тиіс.

## Пакеттік түрлендірудегі анық емес индекс

Профильсіз анық емес DB **«Баптау қажет»** күйін алады. Жолды таңдап, **«Таңдалған DB-ны баптау…»** батырмасын басыңыз, стандартты импорт терезесінде тереңдік/уақытты тағайындаңыз және жоспарды пакеттік операцияға қолданыңыз. Жоспар ағымдағы сеанста сақталады және тек осы файлды қайта іске қосуға болады.
