# Планшет өзара әрекетінің ООП-архитектурасы

## Мақсаты

Архитектура салуды енгізу оқиғаларынан бөледі және мөлдір виджеттің бүкіл планшет редакторын бұғаттауына жол бермейді.

## Компоненттер

### `TabletInteractionRouter`

Қалыптандырылған `TabletInputEvent` оқиғаларын басымдық бойынша өңдегіштерге береді. Логикалық қимылды ұстау күйінің жалғыз иесі — маршрутизатор; native `grabMouse()` қолданылмайды.

Басымдық реті:

1. бар аннотация;
2. белсенді аннотация жасау құралы;
3. жолақты таңдау/өңдеу;
4. қисықтар, геологиялық интервалдар және навигацияның қалыпты логикасы.

### `TabletEditModeCoordinator`

F4 инварианттарын бір жерде басқарады: аннотация мен жолақ өңдегіштері бірге қосылады, жолақ тек жасау құралы белсенді кезде уақытша тоқтайды, F4 өшкенде құрал міндетті түрде алынып, курсорлар қалпына келеді.

### `AnnotationInteractionHandler`

Таңдау, жасау, жылжыту, өлшем өзгерту, екі рет шерту, F2/Enter/Delete және контекст әрекеттерін басқарады. Өзі сурет салмайды. Геометрия модельге тінтуір жіберілгеннен кейін бір рет жазылады.

### `TrackEditInteractionHandler`

Бір шерту жолақты таңдайды, бірақ қисықты таңдауға оқиғаны әрі қарай өткізеді. Екі рет шерту қолдау көрсетілетін жолақтың толық редакторын ашады.

### `TabletAnnotationOverlay`

Тек салу және hit-test орындайды. `WA_TransparentForMouseEvents` тұрақты қосылған, mouse handler, widget mask және pointer grab жоқ. Жылжыту/өлшем өзгерту кезінде тек нысанның dirty-аймағы қайта салынады.

### `TabletInteractionWatchdog`

Логикалық қимыл кезінде сол жақ батырманың нақты күйін тексереді. Windows Alt+Tab, модальды терезе немесе монитор ауысуында release оқиғасын жоғалтса, watchdog қимылды қауіпсіз аяқтап, маршрутизатор күйін тазартады.

## Кеңейту ережелері

Жаңа құрал маршрутизатор өңдегішінің интерфейсін іске асырады. Ол бүкіл canvas үшін бәсекелес event filter орнатпауы, native pointer capture қолданбауы және басқа құралдың күйін тікелей өзгертпеуі тиіс. Режимдер координатор арқылы ауысады.

## Инвариант тесттері

Тесттер paint-only overlay, native capture/mask жоқтығын, бір drag үшін бір commit, жолақ оқиғасының pass-through тәртібін, бар аннотацияның басымдығын және терезе деактивациясы/жоғалған release кезінде күйдің тазалануын тексереді.
