# 0.7.29 шығарылымы — import jobs бірыңғай шекарасы

## Өзгерістер

- `ImportJobController` Qt пакетінен `services/import_jobs.py` файлына көшірілді;
- CSV/Excel, пакеттік LAS және Paradox тіркеуі үшін бірыңғай `DatasetImportJobExecutor` жасалды;
- LAS strict/compatible/manual саясаты, диагностика, қолмен өткізу және кері тереңдікті анықтау
  енді `MainWindow` ішінде орындалмайды;
- lossless LAS source document және import report бір project-session port арқылы тіркеледі;
- Paradox dataset пен жеке ұңғыма фондық импорт сәтті аяқталғаннан кейін сол шекара арқылы қосылады;
- тоқтатылған, қабылданбаған немесе қате файл жартылай тіркелген dataset қалдырмайды;
- жүктелген LAS санағы енді тек нақты сәтті тіркелген файлдарды көрсетеді;
- LAS адаптері lazy түрде жүктеледі, сондықтан routing және CSV/Excel jobs модульді импорттау
  кезінде `lasio` тәуелділігін талап етпейді.

## Үйлесімділік

Жоба форматы, планшет форматы, LAS/DB файлдары және пайдаланушы әрекеттерінің реті өзгерген жоқ.
Бастапқы файлдар бұрынғыдай тек оқу режимінде ашылады.

## Тексеру

Import jobs, адаптерлер және project-session boundary үшін кеңейтілген регрессиялық жиын:
105 тест өтті, 3 платформалық тест өткізілді; `compileall` және wheel жинағы қатесіз аяқталды. Толық Ruff/mypy/pytest gate барлық тәуелділігі орнатылған ортада қайта орындалуы тиіс.
