# Import Review

## Мақсаты

Import Review — CSV/TXT, Excel, LAS және GeoScape/Paradox деректерін жобаға тіркеу алдындағы
міндетті ортақ кезең. Формат адаптері алдымен уақытша `Dataset` жасайды, содан кейін
пайдаланушы индекс, арна семантикасы, өлшем бірліктері, NULL және QC нәтижелерін тексереді.
`ProjectSession` ішіне тек расталған көшірме қосылады.

## Жұмыс тәртібі

1. Дереккөзді таңдап, форматқа арналған параметрлерді толтыру.
2. Файлдың уақытша домендік модельге оқылуын күту.
3. Белсенді индекс пен арналар кестесін тексеру.
4. Қажет болса mapping, UOM немесе қосымша NULL белгісін өзгерту.
5. Бұғаттайтын қателерді түзетіп, «Импортты растау» батырмасын басу.

Терезені жабу немесе бас тарту well/dataset қоспайды және жобаның `dirty` күйін өзгертпейді.

## Индекс және NULL

Review уақытша dataset ішіндегі дайын индекстердің бірін таңдауға, оның мнемоникасын, рөлін,
түрін және өлшем бірлігін өзгертуге мүмкіндік береді. Үйлесімді жұптар:

- `depth` — MD, TVD немесе TVDSS;
- `time` — relative time немесе datetime;
- `generic` — generic index.

`-999.25` сияқты қосымша сандық NULL белгісі тек қабылданған көшірмеде `NaN` мәніне
ауыстырылады. Бастапқы файл мен уақытша dataset өзгермейді.

## Арналарды қолмен сәйкестендіру

Әр арнаны импортқа қосуға/алып тастауға, canonical mnemonic, canonical kind, quantity class
және UOM мәндерін түзетуге немесе Semantic Channel Dictionary автоматты нәтижесін қайтаруға
болады.

Review ішіндегі UOM өзгерісі мәндерді түрлендірмейді, тек метадеректерді түзетеді және
`SemanticChannelBinding.evidence` ішінде жазылады. Физикалық мағына қолмен өзгерсе,
`matched_by=manual_import_review`, confidence `1.0` қолданылады.

## QC және растау ережелері

Алдын ала тексеру индекстегі жарамды/NULL мәндерді, қайталануларды, үзілістерді, аралас немесе
кему ретін, unresolved арналарды, жоқ/белгісіз UOM, UOM мен quantity class қайшылығын,
толығымен бос арналарды және duplicate canonical kind жағдайларын көрсетеді.

Индекс қатесі, UOM/quantity class қайшылығы немесе импортталатын арнаның болмауы растауды
бұғаттайды. Ескертулерді пайдаланушы саналы түрде қабылдай алады.

## Атомарлық шекара

`ImportReviewController.preview()` және `commit()` терең көшірмемен жұмыс істейді. `commit()`
тексеруден кейін ғана жаңа dataset қайтарады, ал `DatasetImportJobExecutor` оны project-session
port арқылы бір командамен тіркейді. Сондықтан бас тарту, қате немесе терезені жабу жоба ішінде
жартылай дерек қалдырмайды.

Қабылданған dataset ішінде `IMPORT_REVIEW_VERSION=1` және `IMPORT_REVIEW_ACCEPTED=true`
сақталады; қосымша sentinel қолданылса, `IMPORT_REVIEW_NULL_VALUE` де жазылады.

## Үйлесімділік

- project format v16 болып қалады;
- LAS/CSV/Excel/DB бастапқы файлдары тек оқу режимінде қалады;
- lossless LAS document және import report review-көшірмеден бөлек сақталады;
- LAS batch әр файл үшін жеке review ашады;
- өлшем бірлігін өзгерту жасырын конверсия орындамайды.
