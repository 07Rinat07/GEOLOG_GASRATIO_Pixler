# Import Review

## Назначение

Import Review является обязательной общей стадией перед регистрацией CSV/TXT, Excel, LAS и
GeoScape/Paradox в проекте. Адаптер сначала создаёт временный `Dataset`, затем пользователь
проверяет индекс, семантику каналов, единицы, NULL и QC. В `ProjectSession` попадает только
подтверждённая копия.

## Рабочий процесс

1. Выбрать источник и параметры специализированного диалога формата.
2. Дождаться разбора файла во временную доменную модель.
3. В Import Review проверить активный индекс и таблицу каналов.
4. При необходимости изменить mapping, UOM или дополнительный NULL-сентинел.
5. Устранить блокирующие ошибки и нажать «Подтвердить импорт».

Отмена закрывает review без добавления скважины/dataset и без изменения `dirty`.

## Индекс и NULL

Review позволяет выбрать один из уже созданных индексов и изменить его мнемонику, роль,
тип и единицу. Поддерживаются согласованные пары:

- `depth` — MD, TVD или TVDSS;
- `time` — relative time или datetime;
- `generic` — generic index.

Дополнительный числовой NULL-сентинел, например `-999.25`, заменяется на `NaN` только в
принятой копии. Исходный файл и временный dataset не изменяются.

## Ручное сопоставление каналов

Для каждого канала можно:

- включить или исключить его из импорта;
- исправить canonical mnemonic и canonical kind;
- выбрать quantity class;
- исправить подпись UOM;
- вернуть автоматическое сопоставление Semantic Channel Dictionary.

Изменение UOM в review не выполняет пересчёт значений. Оно исправляет только метаданные и
фиксируется в `SemanticChannelBinding.evidence`. При ручной смене физического смысла binding
получает `matched_by=manual_import_review` и confidence `1.0`.

## QC и блокировка подтверждения

Предварительный просмотр показывает:

- количество валидных и NULL-значений индекса;
- дубликаты, разрывы, смешанный или убывающий порядок;
- unresolved semantic channel;
- отсутствующую/неизвестную UOM;
- конфликт UOM и quantity class;
- полностью пустые каналы;
- повторяющиеся canonical kinds.

Ошибки индекса, конфликт UOM/quantity class и отсутствие импортируемых каналов блокируют
подтверждение. Предупреждения можно принять осознанно.

## Атомарность

`ImportReviewController.preview()` и `commit()` работают с глубокой копией. `commit()`
возвращает новый dataset только после повторной проверки. `DatasetImportJobExecutor`
регистрирует эту копию одной командой через project-session port. Поэтому ошибка, отмена или
закрытие окна не оставляют частичных данных в проекте.

Принятый dataset содержит служебные параметры `IMPORT_REVIEW_VERSION=1` и
`IMPORT_REVIEW_ACCEPTED=true`; при использовании дополнительного sentinel также сохраняется
`IMPORT_REVIEW_NULL_VALUE`.

## Совместимость

- текущий project format v20 сохраняется;
- исходные LAS/CSV/Excel/DB открываются только для чтения;
- lossless LAS document и import report сохраняются отдельно от review-копии;
- batch LAS открывает отдельный review для каждого файла;
- изменение единицы не конвертирует значения автоматически.

## Диагностика и восстановление

Начиная с 0.7.46 предупреждения остаются неблокирующими, а ошибки собираются по этапам чтения,
разбора, policy, review, регистрации и отображения. Диалог отчёта показывает код, severity,
источник, рекомендуемое действие и технический traceback; отчёт можно скопировать или сохранить.
Если dataset уже зарегистрирован, но планшет не построился, программа открывает безопасную LAS-таблицу,
не удаляя импортированные данные. Повторяющиеся vendor-мнемоники читаются по физическим столбцам,
а повреждение одного канала исключает только этот канал с предупреждением.
## Исправимый смешанный порядок DB

Начиная с 0.7.47 смешанный порядок числового индекса можно исправить флажком «Сортировать
принятую копию по активному индексу». Для GeoScape/Paradox рекомендация включается
автоматически. Все индексы, глубина и кривые переставляются одной стабильной перестановкой.
Исходный DB и временный dataset остаются неизменными. Review показывает информационное
событие `index-sorted-copy`; дубликаты и разрывы остаются предупреждениями.
