# Единая модель печатных форматов

Статус: реализовано в версии 0.7.38. Runtime schema печатной геометрии: v1.
Формат проекта остаётся v16.

## Назначение

Модель отделяет физический носитель, масштаб содержимого и продолжение страниц от Qt-диалогов.
Preview, PDF, постраничный PNG/JPEG/TIFF/BMP/WebP/SVG и системный принтер используют один
`PrintJobSettings` и один `PrintDocumentPlan`.

## Носители

- **A4** — 210 × 297 мм;
- **A3** — 297 × 420 мм;
- **Custom** — ширина и высота 25–5000 мм;
- **Roll** — заданная ширина и вычисляемая длина, не более 5000 мм на сегмент.

Для стандартного и пользовательского листа поддерживаются portrait/landscape. Для рулона ширина
задаётся пользователем, а длина вычисляется по физической высоте содержимого и полям. Более длинный
результат представляется детерминированными страницами/сегментами продолжения.

## Масштаб

### Fit

Содержимое помещается в одну горизонтальную область печати. Для планшетных форм можно дополнительно
включить выравнивание ширины колонок. Вертикальный интервал и число вертикальных страниц задаются
общей `ReportDefinition` и pagination settings.

### 100%

Один логический пиксель исходной горизонтальной геометрии соответствует `1/96` дюйма. Ширины
колонок не сжимаются. Если носитель уже формы, создаются горизонтальные страницы продолжения.
Вертикальная шкала по-прежнему управляется выбранным инженерным интервалом на страницу, поэтому
100% фиксирует ширину формы, а не скрытно изменяет глубинный диапазон.

Перекрытие продолжений задаётся в миллиметрах от 0 до 50 и переводится в reference pixels только
в headless-модели.

## Двумерный план страниц

`build_document_plan()` строит декартово произведение:

1. вертикальных интервалов отчёта;
2. горизонтальных продолжений 100%.

Порядок стабилен: для каждого вертикального интервала последовательно выводятся все горизонтальные
части. Номер страницы глобальный; в шапке указывается номер продолжения. PDF и принтер создают один
многостраничный документ, растровые форматы и SVG — нумерованные файлы.

Системный Print Dialog получает диапазон `1…N`. Если пользователь выбирает часть диапазона, gate и
итоговый счётчик используют только фактически выбранные страницы.

## Physical Printer Gate

После выбора устройства приложение повторно применяет собственные настройки страницы и проверяет:

- валидность принтера и состояние драйвера;
- поддержку A4/A3 либо custom/roll media;
- минимальный и максимальный физический размер custom media;
- минимальные аппаратные поля в миллиметрах;
- фактическую printable area;
- доступные разрешения и ближайший допустимый DPI;
- наличие страниц в плане и многостраничную подачу.

Ошибка блокирует отправку задания. Замена DPI и необходимость непрерывной подачи для нескольких
страниц являются предупреждениями и записываются в журнал.

## Windows gate tool

Список устройств:

```powershell
.\.venv\Scripts\python.exe tools\physical_print_gate.py
```

Проверка без печати:

```powershell
.\.venv\Scripts\python.exe tools\physical_print_gate.py `
  --printer "Имя принтера" --format a3 --scale actual_size
```

Реальный тест отправляется только с явным флагом:

```powershell
.\.venv\Scripts\python.exe tools\physical_print_gate.py `
  --printer "Имя принтера" --format roll --width-mm 300 `
  --scale actual_size --print-test
```

Инструмент не печатает автоматически без `--print-test`.

## Паспорт и ограничения

Report Passport schema v4 подписывает `scale_mode`, overlap продолжений, физический формат, поля и
запрошенный DPI. Файловые результаты получают sidecar; физическая печать показывает digest без
sidecar.

Headless gate не заменяет аппаратную приёмку. Перед stable обязательны Windows-проверки A4/A3,
custom/roll на поддерживаемом plotter, Fit/100%, page range, HiDPI preview, PDF и реальная подача
всех страниц продолжения.
