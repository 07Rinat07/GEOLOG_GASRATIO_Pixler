# Импорт и диагностика LAS

## Текущее безопасное поведение

При открытии LAS приложение сначала читает данные без изменения исходного файла и формирует
структурированный отчёт. Для источника фиксируются путь, размер, SHA-256, объявленная версия
LAS, `WRAP` и числовое значение `NULL`.

Параллельно создаётся `LosslessLasDocument`. Он хранит точные исходные байты и индексирует
секции без переформатирования содержимого. Для каждой секции известны исходный заголовок,
нормализованное имя и байтовые границы. Документ определяет UTF-8/BOM, Windows-1251,
DOS CP866, KOI8-R, Mac Cyrillic и Latin-1 по читаемости заголовка, а также стиль перевода строк.
Вызов `to_bytes()` всегда возвращает исходное содержимое без изменений.

После декодирования приложение восстанавливает распространённые искажения текста, включая
UTF-8→Windows-1251 и CP866→Windows-1251. Исправление применяется к названиям, описаниям,
единицам и мнемоникам только в Unicode-представлении; исходные байты lossless-документа не
перезаписываются. Та же нормализация выполняется при открытии старого проекта, поэтому ранее
сохранённые искажённые подписи становятся читаемыми без изменения исходного LAS.

Отчёт проверяет:

- LAS 1.2/2.x либо режим ограниченной совместимости для другой версии;
- направление, равномерность и номинальный шаг индекса;
- пропуски, дубликаты и разрывы индекса;
- повторяющиеся мнемоники кривых;
- состав и порядок секций фиксируются; отсутствие данных и повторения диагностируются;
- отсутствие секции ASCII и смешанные переводы строк;
- соответствие `STRT`, `STOP` и `STEP` фактическим данным;
- наличие распознаваемого числового `NULL`.

Предупреждения отображаются после загрузки. Убывающая глубина допускается и показывается
отдельно: оригинал остаётся без изменений, а исправление выполняется только созданием новой
возрастающей копии.

## Семантическое распознавание параметров

Порядок кривых в LAS не используется для определения их смысла. После чтения каждой кривой
программа сопоставляет исходную мнемонику, описание и единицу со встроенным Sensors-каталогом и
контролируемым набором распространённых алиасов. Исходная мнемоника сохраняется, а
`canonical_mnemonic` заполняется только при достаточно уверенном результате.

Поддерживаются различия регистра, пробелов, дефисов и подчёркиваний, химические формулы
`CH4/C2H6/C3H8`, русские/казахские/английские описания и визуально одинаковые кириллические и
латинские буквы, например `С1` → `C1`. Если несколько кривых одинаково уверенно соответствуют
одному параметру, автоматический выбор блокируется как неоднозначный. Подробности:
[RU](ru/LAS_PARAMETER_RESOLUTION.md), [KK](kk/LAS_PARAMETER_RESOLUTION.md),
[EN](en/LAS_PARAMETER_RESOLUTION.md).

Для Gas Ratio концентрации `%`, `ppm`, `ppb` и доли приводятся к общей шкале, когда преобразование
однозначно. Несовместимые единицы не смешиваются скрытно.

## Режимы принятия файла

Перед выбором LAS приложение запрашивает политику импорта:

- **совместимый** — блокирует ошибки, открывает файл с предупреждениями и ничего не исправляет
  скрытно;
- **строгий** — блокирует как ошибки, так и любые предупреждения отчёта;
- **ручная проверка** — блокирует ошибки, а при предупреждениях показывает их до добавления
  dataset и требует явного подтверждения.

Все режимы используют один результат парсинга и один import report. Поэтому смена политики
не меняет распознанные значения, индексы или кривые. Если пользователь отклоняет файл при
ручной проверке, он пропускается, а оригинал и проект не изменяются.

## Хранение вместе с проектом

При сохранении проекта lossless-документы записываются в соседний каталог
`<имя-проекта>.assets`. Имя каждого LAS является его SHA-256, поэтому одинаковый исходник не
дублируется. JSON содержит только относительный путь, размер и контрольную сумму. При открытии
путь проверяется от обхода каталогов, а содержимое — по размеру и SHA-256. Отсутствующий или
повреждённый artifact вызывает понятную ошибку вместо незаметной подмены источника.

Манифест source artifacts появился в формате проекта 5. Текущий формат проекта 15 сохраняет
эту модель и последовательно мигрирует старые документы; для legacy-проекта создаётся пустой
манифест. При Save As lossless-источники копируются в каталог нового проекта.

Начиная с формата проекта версии 7, структурированный import report также сохраняется в JSON
как provenance dataset. После повторного открытия Data Inspector восстанавливает исходную
версию/WRAP/NULL, кодировку, список секций, диагностику индекса и issues. При наличии `.assets`
размер и SHA-256 отчёта обязаны совпасть с lossless-файлом; несоответствие блокирует загрузку.

## Экспорт изменённой копии

Если dataset связан с lossless-источником, обычный экспорт новой LAS-копии работает в
составном режиме:

1. Из текущих метаданных, глубины и кривых создаются новые стандартные секции.
2. В исходном документе однозначно заменяются VERSION, WELL, CURVE, PARAMETER и ASCII.
3. Преамбула, пользовательские секции и комментарии переносятся побайтно.
4. Новые секции приводятся к исходной кодировке и стилю перевода строк.
5. Результат атомарно заменяет временный файл; оригинальный LAS не изменяется.

При повторяющихся целевых секциях, отсутствии VERSION/WELL/CURVE/ASCII или невозможности
представить новые данные в исходной кодировке операция завершается ошибкой. Программа не
выбирает неоднозначную секцию и не переключается на потенциально теряющий данные режим молча.

Перед выбором целевого файла открывается `ExportPlan`. Пользователь задаёт:

- LAS 1.2 или LAS 2.0;
- `WRAP=YES/NO`;
- конечное числовое значение `NULL`;
- 1–15 знаков после запятой;
- переносить ли пользовательские секции lossless-источника.

Предварительный анализ блокирует временной индекс без TIME→DEPTH mapping, пустой или
некорректный индекс, бесконечные значения кривых, совпадение `NULL` с реальными данными и
неоднозначную структуру source sections. `NaN` в кривых явно сообщаются как значения, которые
будут заменены выбранным `NULL`. LAS 1.2 сопровождается предупреждением о legacy-совместимости.
Ошибки показываются до диалога выбора файла, предупреждения требуют подтверждения.

Профили LAS 3.0 и автоматический выбор исходной версии остаются следующим этапом матрицы
совместимости.

## Профиль LAS-источника

После импорта вкладка «LAS-источник» в Data Inspector показывает путь, версию LAS, WRAP,
NULL, кодировку, переводы строк, размер, SHA-256, список секций и количество сообщений каждого
уровня. Статус lossless-артефакта «проверен» означает, что его размер и SHA-256 совпадают с
сохранённым import report; отсутствие или несовпадение отображается явно.

Версия 1.2/2.0, WRAP и NULL источника предлагаются как начальные параметры экспорта. Это не
изменяет оригинал и не запускает экспорт автоматически: значения остаются видимыми и
редактируемыми в `ExportPlan`, а результат сохраняется в новый файл.

## Границы текущего этапа

Во время активной сессии исходные байты находятся в памяти. Для очень больших LAS следующим
улучшением станет ленивый файловый handle или memory mapping.

Текущий импорт остаётся совместимым: `import_las()` возвращает `Dataset`, а новый API
`import_las_with_report()` возвращает dataset вместе с диагностикой. Исправления файла не
выполняются скрытно.

5. Ленивое чтение больших source artifacts.
