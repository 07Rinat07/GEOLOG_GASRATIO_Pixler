# Импорт и диагностика LAS

## Текущее безопасное поведение

При открытии LAS приложение сначала читает данные без изменения исходного файла и формирует
структурированный отчёт. Для источника фиксируются путь, размер, SHA-256, объявленная версия
LAS, `WRAP` и числовое значение `NULL`.

Параллельно создаётся `LosslessLasDocument`. Он хранит точные исходные байты и индексирует
секции без переформатирования содержимого. Для каждой секции известны исходный заголовок,
нормализованное имя и байтовые границы. Документ также определяет UTF-8/BOM/CP1251 и стиль
перевода строк. Вызов `to_bytes()` всегда возвращает исходное содержимое без изменений.

Отчёт проверяет:

- LAS 1.2/2.x либо режим ограниченной совместимости для другой версии;
- направление, равномерность и номинальный шаг индекса;
- пропуски, дубликаты и разрывы индекса;
- повторяющиеся мнемоники кривых;
- состав и порядок секций фиксируются; отсутствие данных и повторения диагностируются;
- отсутствие секции ASCII и смешанные переводы строк;
- соответствие `STRT`, `STOP` и `STEP` фактическим данным;
- наличие распознаваемого числового `NULL`.

Предупреждения отображаются после загрузки. Убывающая глубина допускается и показывается
отдельно: оригинал остаётся без изменений, а исправление выполняется только созданием новой
возрастающей копии.

## Хранение вместе с проектом

При сохранении проекта lossless-документы записываются в соседний каталог
`<имя-проекта>.assets`. Имя каждого LAS является его SHA-256, поэтому одинаковый исходник не
дублируется. JSON содержит только относительный путь, размер и контрольную сумму. При открытии
путь проверяется от обхода каталогов, а содержимое — по размеру и SHA-256. Отсутствующий или
повреждённый artifact вызывает понятную ошибку вместо незаметной подмены источника.

Формат проекта версии 5 автоматически совместим со старыми проектами: миграция создаёт пустой
манифест source artifacts. При Save As lossless-источники копируются в каталог нового проекта.

## Границы текущего этапа

Lossless-документ пока не участвует в экспорте изменённой LAS-копии. Неизвестные секции и
комментарии сохраняются в исходном artifact, однако безопасное точечное изменение известных
элементов и перенос остальных секций в новый LAS будут добавлены следующим этапом.

Во время активной сессии исходные байты находятся в памяти. Для очень больших LAS следующим
улучшением станет ленивый файловый handle или memory mapping.

Текущий импорт остаётся совместимым: `import_las()` возвращает `Dataset`, а новый API
`import_las_with_report()` возвращает dataset вместе с диагностикой. Исправления файла не
выполняются скрытно.

## Следующие шаги

1. Строгий, совместимый и ручной режимы чтения.
2. Безопасная замена известных элементов с сохранением остальных секций.
3. Матрица поддержки LAS 1.2, 2.0 и профилей LAS 3.0.
4. Диалог отчёта с выбором и протоколом исправлений.
5. Анализ потерь до версионного экспорта.
6. Ленивое чтение больших source artifacts.
