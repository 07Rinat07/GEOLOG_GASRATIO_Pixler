# Roadmap

Roadmap следует зависимостям продукта, а не номерам экранов. Подробные требования и
статусы находятся в `docs/REQUIREMENTS.md`.

## Реализованный фундамент

- **0.1–0.2 Core/Data** — проект, LAS, NumPy-модель кривых и атомарное сохранение.
- **0.4–0.5 Tablet** — многотрековый планшет, общая глубина, layout и инспектор.
- **0.6 Interactive Editing** — карандаш, Undo/Redo и безопасный LAS-экспорт.
- **Geology foundation** — литология, каталог пород, паттерны, описания, легенда и дерево.
- **Architecture** — repository boundary, миграции проекта и Plugin API.

## Приоритеты после отраслевого аудита

1. Готов lossless-фундамент и `ExportPlan` LAS 1.2/2.0; далее — матрица профилей LAS 3.0 и
   автоматический выбор исходной версии.
2. Готовы несколько индексов MD/TVD/TVDSS/TIME; далее — семантические типы каналов.
3. Версионный экспорт LAS 1.2/2.0/поддерживаемых профилей 3.0 с анализом потерь.
4. Общий SelectionModel таблицы и графика, затем рабочее место сращивания/ресэмплинга.
5. Единый command stack, граф зависимостей формул, UOM и словарь свойств PWLS.
6. Пользовательские профили и личная библиотека после стабилизации схем формул/шаблонов.
7. DLIS/RP66 как изолированный адаптер; WITSML 2.1/ETP 1.2 как сетевой коннектор.

WITSML не реализуется внутри LAS-парсера. Его разработка начинается после нескольких
индексов, типизированных каналов, UOM и устойчивого локального хранилища.
